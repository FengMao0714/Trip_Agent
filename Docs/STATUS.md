## 2. P0 验收 Checklist

从 PRD 中提取的所有 P0 功能验收条件：

### 对话交互

- [ ] 支持中文自由文本输入，长度不限
- [ ] 系统能从自由文本中提取出目的地、天数、预算、旅行风格、饮食偏好
- [ ] 缺失关键字段时 Agent 主动追问（P1，但建议纳入）
- [ ] Agent 回复以 SSE 流式方式推送到前端
- [ ] 首字符延迟 (TTFT) < 2s
- [ ] 输出过程流畅无卡顿

### 智能规划引擎

- [ ] Agent 能自主决定何时调用工具、何时生成回复
- [ ] ReAct 循环次数上限 10 次，超过自动终止
- [ ] 从对话中提取的偏好信息在会话内持久化
- [ ] 输出 JSON 格式行程，包含：日期、时段、地点、活动、交通方式、费用估算
- [ ] 3 天行程完整生成时间 < 30s

### RAG 知识库

- [ ] 知识库覆盖至少 3 个热门城市
- [ ] 每城市不少于 50 个景点/餐厅条目
- [ ] Agent 生成的行程中，景点信息与知识库一致，幻觉率显著降低

### 外部工具调用

- [ ] 高德 POI 搜索：返回名称、地址、经纬度、评分
- [ ] 高德路径规划：返回距离 (km) 和预计耗时 (分钟)
- [ ] MCP Server 正常运行，Agent 可自主调用
- [ ] MCP 不可用时，能降级为直接 API 调用

### 前端展示

- [ ] 对话式 UI：支持消息输入、发送、历史消息展示、流式渲染
- [ ] 行程按天分组，每个景点显示名称、时间、交通方式
- [ ] 在 1280px 以上分辨率下布局正常
- [ ] 页面首屏加载 < 3s

### 端到端验收场景

- [ ] **场景 1**：输入"北京3天，预算3000，历史文化，不吃辣" → 生成完整行程 + 地图标注
- [ ] **场景 2**：在场景 1 基础上说"换掉第二天下午的景点" → 局部修改生效（P1）
- [ ] **场景 3**：输入复杂约束"带老人去成都4天" → 行程节奏合理（加分项）

## 3. 风险登记表

| R4 | **RAG 检索质量差** | 返回无关知识片段，行程质量下降 | 中 | 🔴 未验证 | 精心设计 chunk 策略；Hybrid Search；RAGAS 持续评估 |
