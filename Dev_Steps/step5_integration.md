## 5.1 前端对接真实 API

```
目标：将前端 Mock 数据替换为真实后端 API 调用。

操作：
1. 更新 src/lib/api.ts：
   - BASE_URL 指向 http://localhost:8000
   - 实现 fetchChat() 真实 POST /api/v1/chat

2. 更新 src/lib/sse.ts：
   - 对接真实 SSE 流
   - 处理所有 7 种事件类型

3. 更新 src/hooks/useChat.ts：
   - 移除 Mock 逻辑，使用真实 API
   - 错误处理：429 限流提示、503 服务不可用、网络错误

4. 确保 CORS 配置正确（后端允许 localhost:3000）

约束：
- 保留 Mock 模式开关（环境变量 NEXT_PUBLIC_USE_MOCK=true）
- 所有 fetch 有 try-catch

完成标准：
- 前端发消息 → 后端 Agent 处理 → SSE 流式回复 → 前端渲染
- 流式输出流畅，无卡顿
```

## 5.2 行程 JSON 解析 + 渲染

```
目标：前端正确解析后端返回的行程 JSON 并渲染卡片。

操作：
1. 更新 parseItinerary.ts：
   - 从 SSE itinerary 事件中提取行程数据
   - 类型校验和容错处理
   - 字段缺失时提供默认值

2. 更新行程组件：
   - TripSummaryCard 显示真实目的地/天数/预算
   - DayPlanCard 渲染真实活动数据
   - BudgetSummary 计算真实费用

约束：
- JSON 解析失败不崩溃，显示友好错误
- 部分数据缺失时降级显示

完成标准：
- 真实行程数据正确渲染在卡片中
- 数据异常时不白屏
```
