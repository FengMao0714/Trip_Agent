# 题   目:  面向个性化行程规划的智能Agent旅游助手的	

## 1.课题的意义   
在当前旅游业数字化与个性化需求日益增长的背景下，用户规划旅行时面临信息碎片化、方案雷同、动态调整困难等问题。虽然存在各类旅行预订App，但其功能多集中于“预订”而非“智能规划”，无法像真人助手一样理解用户复杂意图并提供一站式定制方案。近年来，大语言模型与AI Agent技术的突破，为解决此问题提供了新路径。本课题旨在设计并实现一个基于多智能体协作的个性化行程规划系统，通过模拟“需求分析-规划-资源查询-预算管理”的人类规划流程，为用户生成高效、合理、可执行的旅行方案。该课题综合运用人工智能、软件工程、前后端开发等技术，实践性强，紧跟当前技术发展趋势，具备良好的学术探索价值与实际应用前景。
## 2.国内外研究现状
目前，主流的在线旅游平台（如携程、Booking.com）其行程规划功能多基于固定模板或简单的筛选组合，智能化程度有限，难以处理复杂的个性化约束。在学术界与工业界，基于AI的旅行推荐已成为研究热点。早期研究多集中于协同过滤、内容推荐等传统算法，但难以实现跨多要素（食、住、行、游）的协同规划。近年来，随着大语言模型的涌现，研究者开始探索利用其强大的语义理解与生成能力进行行程规划。例如，国外有研究利用LLM进行旅游对话系统构建，国内如百度、阿里等公司也推出了结合大模型的智能导览雏形。然而，将LLM作为核心推理引擎，构建一个具备自主任务分解与工具调用能力的多智能体（Multi-Agent）旅游助手系统，仍是一个处于前沿探索阶段的课题。本课题旨在此方向进行工程化实践，设计一个可运行的智能体协作框架。
## 3.主要内容
(1)用户偏好画像构建：基于语义理解用户预算、风格及饮食偏好。
(2)基于 RAG 的垂直知识库：构建目的地景点、规则的向量数据库。
(3)ReAct 智能规划引擎：利用 Agent 的“思考-行动-观察”循环完成行程排列。
(4)基于MCP协议的外部工具集成：摒弃传统的手写 API 封装，引入 Model Context Protocol (MCP) 标准，以标准化的 Server 模式接入高德地图（POI 搜索、路径计算）及天气预报，实现跨平台工具的无缝调用。
(5)交互式行程微调：支持用户在生成后通过对话对局部行程进行重绘或增删。
(6)可视化前端展示：基于 React/Next.js 实现流式渲染及地图打点交互。
## 4.方法与步骤
(1)环境搭建：使用 Python FastAPI 作为后端框架，Next.js 搭建前端。
(2)架构设计：采用 LangGraph 管理 Agent 状态机，确保长对话上下文的一致性。
(3)数据准备：利用 pgvector 存储旅游知识切片，实现 Hybrid Search 混合检索。
(4)标准化工具调用链实现：结合 langchain-mcp-adapters 与高德 MCP Server，实现 Agent 工具链的即插即用，大幅降低由于 API 接口变更导致的系统耦合风险。
(5)系统优化：引入 Stream 流式传输提升响应速度，并对高频查询设计 Redis 缓存方案。
(6)测试评估：通过 RAGAS 评估指标对回答的忠实度与相关性进行定量分析。

## 5.阶段进度计划
1-2周：文献调研，完成系统需求分析与数据库 E-R 图设计。
3-5周：后端核心 Agent 逻辑开发（基于 LangGraph 和 DeepSeek-V3 模型）。
6-8周：RAG 知识库填充与外部 API（高德/Google Maps）联调。
9-10周：前端界面开发、流式交互实现及系统联调。
11-12周：撰写毕业论文，整理代码，准备最终演示 Demo。
